# 端到端技术流程

[English](pipeline.md) | 中文

本文档描述从数据采集到实车执行的完整技术流程。

---

## 流程阶段

```
数据采集 → 数据清洗 → 模型训练 → 模型导出 → Jetson 部署 → MCU 执行
```

---

## 阶段 1：数据采集

| 项目 | 说明 |
|------|------|
| **输入** | 无（人工驾驶） |
| **输出** | 原始数据：图像及对应控制信号 |
| **作用** | 采集人类驾驶行为，用于模仿学习 |

操作者人工驾驶车辆，算力模块同步记录摄像头图像与控制信号（转向、油门、刹车）。这些配对的样本构成训练基础。

由 `vision-follow-car-jetson`（录制）或产出兼容数据的采集方案完成。

---

## 阶段 2：数据清洗

| 项目 | 说明 |
|------|------|
| **输入** | 阶段 1 的原始数据 |
| **输出** | 清洗后数据集：有效图像–控制配对 |
| **作用** | 剔除无效或不一致样本，为训练做准备 |

剔除无效帧（如模糊、缺失控制、时间不对齐等），将剩余数据整理为统一格式供模型训练使用。

由 `vision-follow-car-training` 完成。

---

## 阶段 3：模型训练

| 项目 | 说明 |
|------|------|
| **输入** | 清洗后的数据集 |
| **输出** | 训练得到的模型权重 / 检查点 |
| **作用** | 从人类示范中学习图像到控制的映射 |

训练神经网络从图像输入预测控制输出。训练在具有 GPU 加速的服务器上运行。

由 `vision-follow-car-training` 完成。

---

## 阶段 4：模型导出

| 项目 | 说明 |
|------|------|
| **输入** | 训练得到的模型（检查点） |
| **输出** | 适合 Jetson 推理的 deployable 模型格式 |
| **作用** | 将训练产物转换为推理引擎可加载的格式 |

将模型导出为可在 Jetson 上加载并实时推理的格式（如 ONNX、TensorRT 或框架特定格式）。

由 `vision-follow-car-training` 完成。

---

## 阶段 5：Jetson 部署

| 项目 | 说明 |
|------|------|
| **输入** | 导出模型、实时摄像头流 |
| **输出** | 实时控制指令（转向、油门、刹车） |
| **作用** | 在车端运行推理并输出控制 |

将导出的模型部署到 Jetson Orin Nano。CSI 摄像头图像输入模型，模型输出控制值，并通过 UART 下发给 MCU。

由 `vision-follow-car-jetson` 完成。

---

## 阶段 6：MCU 执行

| 项目 | 说明 |
|------|------|
| **输入** | Jetson 下发的控制指令（UART） |
| **输出** | 物理执行：转向角度、油门、刹车状态 |
| **作用** | 将指令转化为硬件动作 |

STM32 接收指令，转换为 PWM 或其他控制信号，驱动转向机构、油门与刹车电缸。

由 `vision-follow-car-mcu` 完成。

---

## 汇总表

| 阶段 | 输入 | 输出 | 项目 |
|------|------|------|------|
| 数据采集 | 人工驾驶 | 原始图像 + 控制 | Jetson / 采集方案 |
| 数据清洗 | 原始数据 | 清洗后数据集 | vision-follow-car-training |
| 模型训练 | 清洗后数据集 | 训练模型 | vision-follow-car-training |
| 模型导出 | 训练模型 | 可部署模型 | vision-follow-car-training |
| Jetson 部署 | 模型 + 摄像头 | 控制指令 | vision-follow-car-jetson |
| MCU 执行 | 控制指令 | 执行机构信号 | vision-follow-car-mcu |
