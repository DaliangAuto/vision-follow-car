# 系统总体概述

[English](system_overview.md) | 中文

本文档描述 vision-follow-car 项目的整体架构。

---

## 总仓库的作用

`vision-follow-car` 仓库是一个**项目说明与导航枢纽**。它不包含可执行代码，其作用是：

- 项目主页与入口
- 高层系统说明与架构介绍
- 三个子仓库的链接
- 从数据到部署的完整流程文档

---

## 三组件架构

系统拆分为三个独立子项目，分别运行在不同硬件上：

| 组件 | 仓库 | 设备 | 主要职责 |
|------|------|------|----------|
| 训练端 | vision-follow-car-training | 服务器（CPU/GPU） | 数据处理、模型训练、验证、导出 |
| 推理端 | vision-follow-car-jetson   | Jetson Orin Nano | 摄像头采集、多帧缓存、神经网络推理、控制输出 |
| 控制端 | vision-follow-car-mcu      | STM32F103RCT6   | 转向、油门、刹车、模式切换、串口通信 |

---

## 为什么拆成三个子仓库

1. **开发环境不同** — 服务器（Linux/Windows）、嵌入式 Linux（Jetson）、裸机 MCU（STM32）的工具链、构建系统、依赖各不相同。

2. **部署目标不同** — 每个子项目分别部署到各自的平台，保持独立可避免无关代码与配置混杂。

3. **边界清晰** — 各仓库职责明确：训练产出模型；Jetson 消费模型并输出指令；MCU 消费指令并驱动执行机构。

4. **版本独立** — 子项目可以按各自节奏演进，无需强制同步发布。

## 组件间关系

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          离线（训练）                                         │
│  vision-follow-car-training                                                 │
│  原始数据 → 清洗数据 → 训练模型 → 导出模型                                     │
└─────────────────────────────┬───────────────────────────────────────────────┘
                              │
                              │ 导出后的模型
                              v
┌─────────────────────────────────────────────────────────────────────────────┐
│                          在线（推理）                                         │
│  vision-follow-car-jetson                                                    │
│  CSI 摄像头 → 图像缓冲 → 神经网络推理 → 控制指令                               │
└─────────────────────────────┬───────────────────────────────────────────────┘
                              │
                              │ 串口（UART）
                              v
┌─────────────────────────────────────────────────────────────────────────────┐
│                          执行（控制）                                         │
│  vision-follow-car-mcu                                                       │
│  接收指令 → 转向 / 油门 / 刹车                                                │
└─────────────────────────────────────────────────────────────────────────────┘
```

- **训练端 → Jetson**：训练项目导出模型，供 Jetson 项目加载用于推理。

- **Jetson → MCU**：Jetson 通过 UART 下发控制指令，MCU 接收后驱动转向机构、油门与刹车电缸。

- **MCU → Jetson（数据采集时）**：采集训练数据时，MCU 通过串口向算力模块发送车辆控制数据（转向、油门、刹车），Jetson 将其与摄像头帧同步记录，作为训练标签。

---

## 数据流概要

1. **感知**：CSI 摄像头（IMX219）在 Jetson 上采集图像。
2. **推理**：Jetson 对图像帧运行神经网络，输出转向 / 油门 / 刹车数值。
3. **执行**：STM32 通过串口接收数值，控制执行机构。

**数据采集时**：MCU 通过串口向 Jetson 发送控制数据，Jetson 与视频帧一并保存供训练使用。训练流程在离线环境中运行，产出第 2 步使用的模型。
